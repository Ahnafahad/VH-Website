# LexiCore Custom Visual Asset Brief

This document is a production brief for an image-creation agent. Give the agent this file, `DESIGN.md`, the existing LexiCore logo, and the attached visual references. Its job is to create a unified custom visual system that fuses the references’ artistic language with LexiCore’s dark editorial vocabulary-learning identity.

This is not a request for unrelated illustrations in a similar style. Every output must look as though it belongs to one designed product family and must remain usable across mobile, tablet, desktop, dark mode, and light mode.

## 1. Inputs to give the image agent

Attach the following together:

1. `DESIGN.md` — product, interface, color, typography, and screen behavior.
2. This file — visual asset inventory and delivery rules.
3. `public/lexicore-logo.png` — the approved LexiCore L mark. It must not be redesigned.
4. Your style-reference images — these define the artistic treatment to fuse with LexiCore.
5. Optional screenshots of the current LexiCore screens — these clarify the space and context in which assets will appear.

The attached references are inspiration for shape language, texture, lighting, rendering, composition, and emotional tone. They are not permission to copy a protected character, logo, composition, or uniquely identifiable artwork.

## 2. Desired fusion

The final direction must combine:

- The artistic language inferred from the attached references.
- LexiCore’s charcoal-black editorial interface.
- Crimson action and energy.
- Gold reward, knowledge, and earned-value accents.
- The atmosphere of language, memory, study, discovery, and mastery.
- A mature premium tone suitable for university-admission learners.

The result should feel intelligent, collectible, slightly mysterious, and tactile. Avoid a childish school aesthetic, generic corporate vectors, glossy 3D mobile-game art, stock illustrations, and unrelated fantasy decoration.

## 3. Build a visual grammar before individual assets

Before generating final elements, the image agent must define one visual grammar and apply it consistently.

### Shape language

Create recurring forms inspired by:

- The curve and silhouette of the existing L mark.
- Open books, pages, quotation marks, letterforms, and dictionary tabs.
- Memory pathways, neural connections, constellations, and linked nodes.
- Index cards, dossiers, seals, medallions, and archival labels.
- Subtle sparks, rays, rings, and orbit lines for progress and recall.

Choose two or three recurring motifs from this list. Do not use every metaphor in every asset.

### Line and edge language

The reference images should determine whether the family uses etched lines, cut-paper edges, inked contours, geometric forms, soft modeled shapes, or another treatment. Whatever is selected must stay consistent across navigation icons, badges, game art, and empty states.

Small icons must remain readable at 20–24 px. Fine texture may appear in larger illustrations, but critical silhouettes cannot depend on tiny detail.

### Materials and texture

Possible LexiCore-compatible materials include charcoal paper, blackened metal, oxblood enamel, aged gold foil, embossed card stock, soft glass, ink, or restrained luminous energy. Select materials based on the attached references and use them as a system.

Avoid making every object shiny. The interface itself is matte; highlights should feel earned.

### Lighting

Use a consistent lighting direction. Crimson may provide a faint active-state rim light; gold may create a soft reward glow. Transparent assets must not include a large rectangular haze that becomes visible on the page.

### Color discipline

Core colors:

- Base black: `#0F0F0F`
- Surface charcoal: `#1A1A1A`
- Elevated charcoal: `#242424`
- Crimson: `#E63946`
- Gold: `#F4A828`
- Primary ivory-white: `#F5F5F5`
- Secondary gray: `#B0B0B0`
- Success green: `#2ECC71`
- Study blue: `#5BA3F5`
- Learning orange: `#F97316`
- Strong teal: `#2DD4BF`

Most artwork should be neutral charcoal/ivory with one dominant accent. Do not mix red, gold, blue, green, and teal equally in a single asset.

## 5. Screen-size and composition variants

Large feature illustrations may need these compositions:

| Variant | Ratio | Intended use |
|---|---:|---|
| Icon | `1:1` | 20–32 px navigation and controls |
| Spot | `1:1` | 48–120 px cards and states |
| Mobile hero | `4:3` or `1:1` | 320–390 px screen compositions |
| Desktop hero | `16:9` or `3:2` | Wide card/empty-state areas |
| Sheet header | `3:1` | Bottom-sheet/dialog headers |
| Background motif | Flexible/wide | Edge decoration and ambient framing |

Do not solve responsive design by shrinking a detailed desktop scene. Recompose important hero art for mobile and desktop when the focal point or crop would otherwise fail.

Every composition must define a safe zone for UI copy. The artwork should frame content, not place detail behind it.

## 6. State and theme variants

For each asset, determine whether it needs:

- Dark-background and light-background treatment.
- Neutral and active state.
- Locked and unlocked state.
- Success, warning, and danger state.
- Reduced-detail small-size version.
- Reduced-motion static endpoint.

Do not generate redundant raster files when code can recolor a clean SVG. Use `currentColor` for functional monochrome icons. Generate separate art only when state changes material, lighting, internal structure, or meaning.

## 7. File and delivery requirements

### Preferred formats

- Functional icons and simple ornaments: SVG.
- Badge/emblem art: SVG when feasible; otherwise transparent WebP and source file.
- Complex illustrations: transparent WebP plus editable source.
- Textures: seamless WebP.
- Animated effects: Lottie JSON, Rive, or transparent WebM only after a static design is approved.

### SVG requirements

- Clean viewBox and no unnecessary canvas padding.
- Expanded or bundled shapes with no missing external fonts.
- Use `currentColor` for recolorable functional icons.
- Group meaningful layers and name them clearly.
- Avoid huge embedded raster images.
- Test at the actual smallest display size.

### Raster requirements

- Transparent background unless a full background is specifically requested.
- Export at 2× the maximum intended CSS display size.
- Use WebP with visually lossless settings.
- No baked-in drop shadow that is clipped by the canvas.
- No text, numbers, sample scores, or fake interface controls.

### Editable source

Deliver the original layered source or vector file. Separate foreground object, glow, shadow, particles, and background motifs where possible so the implementation can adapt them.

### Naming

Use lowercase kebab-case and group by feature:

```text
lexicore/
  core/
  navigation/
  home/
  study/
  flashcards/
  quiz/
  practice/
  leaderboard/
  badges/
  games/word-hunt/
  games/word-charge/
  profile/
  onboarding/
  states/
  textures/
```

Examples:

```text
navigation/study.svg
home/recommendation-due-review.webp
mastery/mastered.svg
games/word-hunt/reveal-success.webp
states/offline.webp
badges/streak-30.webp
```

### Manifest

Deliver an asset manifest with:

- Filename.
- Purpose.
- Intended screen/component.
- CSS display size.
- Safe zone.
- Dark/light compatibility.
- Available states.
- Whether recoloring is allowed.
- Alt-text recommendation or `decorative` designation.

## 8. What must remain live UI

Do not turn the product into screenshots. The following must remain HTML/CSS/code rather than baked into images:

- All words, definitions, examples, clues, instructions, labels, and headings.
- Names, avatars, dates, rankings, points, streaks, and progress values.
- Buttons, inputs, checkboxes, toggles, tabs, and focus states.
- Progress-ring values and dynamic progress bars.
- Quiz options and answer explanations.
- Flashcard content.
- Dossier messages.
- Leaderboard rows.
- Locked/unlocked state logic.

Artwork should provide icons, frames, atmosphere, motifs, and illustrations around those live elements.

## 9. Visual consistency checks

Before approving an asset batch, verify:

- It is recognizable without reading a label.
- It still works at its real display size.
- It looks correct on `#0F0F0F`, `#1A1A1A`, `#F7F7F8`, and `#FFFFFF`.
- It uses the same perspective, edge treatment, material, and lighting as the other assets.
- Crimson communicates action or active energy.
- Gold communicates value, reward, or knowledge.
- It does not modify or imitate the L logo incorrectly.
- It contains no accidental letters, gibberish, signatures, watermarks, or baked-in UI text.
- It leaves sufficient negative space for responsive content.
- It adds identity without making the screen noisy.

## 11. Master prompt for the image-creation agent

Use the following as the initial instruction after attaching the references, `DESIGN.md`, this brief, and the existing logo:

> You are the visual design and asset-production agent for LexiCore, a premium vocabulary-learning application. Read the attached `DESIGN.md` and `LEXICORE_VISUAL_ASSET_BRIEF.md` completely before creating anything. Inspect every attached style-reference image and infer its shared shape language, rendering method, texture, lighting, composition, and emotional tone. Fuse those traits with LexiCore’s charcoal editorial interface, crimson action language, gold earned-value language, and themes of vocabulary, memory, recall, discovery, and mastery.
>
> The existing LexiCore L logo is approved and immutable. Use it as supplied; do not redraw, reinterpret, recolor, distort, or replace it. Build a related visual family around it without copying the logo into every asset.
>
> Begin with Batch 1 only: a style tile, six core motifs, four surface textures, the five navigation icon proofs, one Home recommendation illustration, and one empty-state illustration. Do not generate all assets immediately. First establish a coherent grammar that can scale from 20 px icons to large responsive illustrations.
>
> Keep the tone mature, intelligent, collectible, tactile, and slightly mysterious. Avoid childish education art, stock corporate vectors, generic gradients, excessive neon, glossy game UI, unrelated fantasy motifs, and visual clutter. Use neutral charcoal/ivory as the foundation, crimson for active energy, and gold for knowledge or earned value. Preserve strong silhouettes and negative space.
>
> Treat all UI text and data as live interface content. Do not bake words, labels, numbers, buttons, progress values, rankings, clues, or definitions into images. Create flexible frames, transparent ornaments, icons, illustrations, and texture layers that can surround responsive HTML content.
>
> For each proposed asset, state its role, intended display size, dark/light behavior, responsive crop or composition, safe zone, format, and whether code may recolor it. Show functional icons at their true small size. Provide dark and light context previews, but deliver transparent production files without mockup backgrounds.
>
> Do not imitate a protected character, logo, or exact composition from the references. Translate their high-level visual qualities into an original LexiCore system. Maintain consistent lighting, perspective, edge language, material treatment, and detail density across every output.
